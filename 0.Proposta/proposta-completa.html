2 Módulos do Sistema

01. Identity & Ambientes Isolados

  • Autenticação via Keycloak (OpenSource)
  • SSO, OAuth 2.0 e OpenID Connect
  • Ambientes Isolados Multibanco (1 database dedicado por sócio -- isolamento físico total)
  • Controle de Acesso (RBAC)
  • Subdomínio dinâmico por sócio

02. Cadastro de Clientes

  • CRUD completo (CPF/CNPJ)
  • Unidades Consumidoras (UC COELBA)
  • Importação de base existente
  • Histórico de alterações

03. Gestão de Contratos

  • Sem fidelidade / 1 ano / 2 anos
  • Regras de cancelamento
  • Vigência e renovação
  • Multa por quebra contratual

04. Gestão de Geração (entrada manual)

  • Cadastro de usinas solares
  • Registro manual de produção (kWh)
  • Registro de geração via planilha ou digitação
  • Histórico de geração

05. Distribuição de Créditos

  • Modo Percentual (X% por unidade)
  • Modo Prioridade (ordem de abatimento)
  • Créditos excedentes acumulados
  • Engine de rateio automático

06. Faturamento & Cobrança

  • Geração de faturas (sem gateway no Core)
  • Template do cliente preenchido com dados do BD
  • Importação de QR Code, código de barras e print da conta de luz (S3)
  • Envio e baixa manuais
  • Juros: 2% + 0,033%/dia
  • Máquina de estado: Gerada - Enviada - Paga

07. Pagamento Automatizado (Asaas)

  • Emissão de boletos e Pix via API
  • Webhook de baixa automática
  • Conciliação bancária automatizada
  • Padrão do mercado brasileiro

08. Comunicação WhatsApp

  • Envio de faturas e boletos (PDF anexo)
  • Envio de documentos (relatórios, recibos)
  • Mensagens curtas (lembretes, avisos)
  • WhatsApp Cloud API + templates HSM
  • Canal unidirecional -- sem bot conversacional
  • Roadmap futuro: bot via n8n (escopo separado)

8b. Notificações em Tempo Real (SignalR)

  • Push in-app via SignalR (WebSocket)
  • Sino no topbar com badge de não lidas
  • Toast no canto + painel dropdown
  • Eventos: pagamento, faturamento, geração, cobrança, alertas
  • Ambientes Isolados -- cada sócio só recebe os próprios eventos

09. Dashboards & Relatórios

  • Dashboard financeiro
  • Relatório de produção
  • Aging de recebíveis
  • Exportação PDF/Excel

Infra simples, custo baixo

Monolito Modular

Aplicação única organizada em módulos independentes. Deploy simples, infra barata, evolução rápida. Sem complexidade de microserviços — mas pronto pra separar quando justificar.

Manutenção sem surpresas

DDD — Domain-Driven Design

Código organizado por domínio de negócio: "Crédito de Energia", "Rateio", "Beneficiário". Qualquer dev entende o sistema rápido — onboarding barato, manutenção previsível.

Performance que escala

CQRS — Command/Query Separation

NHibernate para escrita (commands) e Dapper para leitura (queries). Dashboards rápidos mesmo com milhares de UCs. Escala sem refatoração.

Orientada a Eventos

AWS SNS + SQS FIFO + Lambda

Cada ação (pagamento confirmado, fatura gerada, crédito distribuído) dispara eventos de domínio com ordem garantida, deduplicação por ambiente+fatura e DLQ. Módulos desacoplados — WhatsApp, Asaas e SignalR reagem automáticamente aos eventos sem dependência direta.

i

Ambientes Isolados: 1 database dedicado por sócio

Cada sócio possui seu próprio banco de dados físico (não schema isolado). Backup, restore e segurança são totalmente independentes entre sócios. A aplicação compartilha a camada de serviço, preservando economia de escala.

Decisão Arquitetural -- Segregação Completa da Infraestrutura?

Cada operação pode rodar em infraestrutura totalmente isolada (banco, app e ambiente próprios) ou compartilhar a mesma plataforma com isolamento físico via ambientes isolados multibanco.

Pontos Positivos da Segregação Total
  • Isolamento absoluto (falha/deploy não propaga)
  • Soberania total: janela de manutenção, versão, backup próprios
  • Compliance simplificado
  • Portabilidade: saída de um sócio não afeta os demais
Pontos Negativos da Segregação Total
  • Custo de infra multiplicado (N ambientes)
  • Manutenção N vezes maior
  • Sem economia de escala
  • Consolidação de indicadores vira projeto a parte
  • Suporte mais caro
Recomendação técnica: Ambientes Isolados Multibanco já entrega isolamento físico total de dados (banco próprio por sócio, backup e segurança independentes), preservando a economia de escala da camada de aplicação. Segregação completa permanece disponível mediante orçamento adicional.

i

Modelo de Precificação Modular

O investimento foi estruturado em 3 blocos independentes para reduzir a barreira de entrada. Um único sócio pode iniciar com um MVP enxuto e adicionar módulos conforme novos parceiros entrem ou a operação cresca.

5.1 Assessment Individual (por sócio)

!

Assessment cobrado a parte para TODOS os sócios

O Assessment é uma fase obrigatória e cobrada a parte para todos os sócios. Não está incluso no valor do Core.

Servico Valor Pagamento

Importante: Operações de média/grande complexidade (múltiplas usinas, integrações com ERP, regras de negócio não-padrão) podem requerer orçamento de desenvolvimento individual, separado e adicional ao Assessment.

7 Pós Go-Live

Evolução e Manutenção Sob Demanda (Sem Recorrência)

Modelo sem amarração e sem mensalidade de suporte. Uma vez entregue o Core em produção, o cliente assume total autonomia sobre a plataforma. A sustentação de infraestrutura (AWS) é provisionada e paga diretamente pelo cliente.

Qualquer intervenção técnica posterior -- manutenções corretivas, integrações adicionais, evoluções funcionais ou novas features -- é tratada como projeto avulso e pontual, orçada sob demanda conforme escopo específico.
