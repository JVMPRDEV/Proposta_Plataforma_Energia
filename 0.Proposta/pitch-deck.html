02 / 19 PARTE 1 · Cenário Atual

Quatro sócios,
quatro realidades distintas

Cada operação cresceu do seu jeito — com seu porte, suas ferramentas e sua maneira de controlar o dia a dia.

Estágios diferentes

De 27 a 120+ UCs, faixas de potência distintas, maturidade operacional diferente. Não existe um único "padrão" entre os sócios.

Ferramentas próprias

Planilhas, ERPs leves, controles próprios. Cada um encontrou um jeito que funciona — para o tamanho atual da sua operação.

Trajetórias independentes

Cada sócio define seu próprio ritmo de captação, sua carteira e sua estratégia comercial. Os caminhos não se sobrepõem.

O ponto de partida não é uma "dor comum" — é a constatação de que cada sócio tem o seu cenário, e qualquer solução precisa respeitar isso.
03 / 19 PARTE 1 · Cenário Atual

O ponto em comum:
o próximo salto de escala

Operações diferentes esbarram, mais cedo ou mais tarde, nos mesmos limites estruturais quando crescem.

Escala

Dobrar de tamanho costuma exigir dobrar a equipe administrativa — a menos que processo e ferramenta evoluam junto.

Padronização

Cada novo cliente reaproveita pouco do anterior. Padronizar reduz retrabalho e libera tempo para vender e atender.

Continuidade

Backup, controle de acesso, histórico auditável e LGPD passam a importar à medida que a base de clientes cresce.

Não é uma urgência para hoje — é a infraestrutura para o próximo ciclo de crescimento de cada sócio.
04 / 19 PARTE 1 · Cenário Atual

Uma plataforma,
ambientes independentes

Cada sócio tem seu próprio ambiente — banco de dados dedicado, dados isolados, operação 100% independente.

Seu ambiente, suas regras

Cada sócio acessa pelo seu próprio subdomínio: seus clientes, suas usinas, suas regras de rateio e seu faturamento. Nada é compartilhado — o isolamento é físico, não apenas lógico.

Mesma plataforma, custo dividido

O código é um só — desenvolvido uma vez, usado por todos. Cada sócio opera isolado, mas o investimento é compartilhado. Mais sócios = menor custo por sócio.

Autonomia total para cada sócio. Custo de desenvolvimento dividido entre todos.

08 / 19 PARTE 3 · Game Changers

O que nenhum SaaS de mercado oferece junto

R$ 0 por UC

Custo zero por UC adicional. Crescer de 27 para 500 UCs não muda o custo. Na Lumi, cada UC custa R$ 6/mês para sempre.

Asaas Nativo

Boleto + Pix + baixa automática via webhook. Conciliação bancária zero esforço. Padrão do mercado de energia.

WhatsApp Oficial

Envio de faturas, boletos e avisos direto pela Cloud API da Meta. Sem WhatsApp pessoal, sem risco de bloqueio.

Eventos em Tempo Real

Arquitetura orientada a eventos (AWS SNS+SQS). Notificações SignalR — pagamento confirmado? Alerta instantâneo.

Propriedade Total

Código, dados e IP são de vocês. Ativo no balanço, não despesa. Nenhum SaaS de mercado permite isso.

Rentabilização

Ofereça a plataforma a outros operadores como white-label ou mensalidade. O software vira fonte de receita.

Seis diferenciais que nenhum modelo de aluguel consegue entregar.
09 / 19 PARTE 4 · Mitigação de Risco

Arquitetura que protege o investimento

Decisões maduras que reduzem custo operacional e garantem que qualquer equipe consiga dar manutenção.

Infra simples, custo baixo

Monolito Modular

Um deploy, uma infra, evolução rápida. Sem a complexidade e o custo de microserviços — mas pronto pra separar quando a operação justificar.

Manutenção sem surpresas

DDD — Domain-Driven Design

O código fala a linguagem de vocês: "Crédito de Energia", "Rateio", "Beneficiário". Qualquer dev entende o sistema rápido — onboarding barato, manutenção previsível.

Performance que escala

CQRS — Command/Query Separation

Leitura separada de escrita. Dashboards rápidos mesmo com milhares de UCs. Escala sem refatoração — a arquitetura já nasceu preparada.

Stack mainstream (.NET 8 + Vue 3 + SQL Server) — qualquer fornecedor sério dá continuidade. Vocês nunca ficam reféns de um time específico.
10 / 19 PARTE 4 · Garantias

Os 4 medos — e como protegemos vocês

"E se atrasar?"

Pagamento 30/30/40 atrelado a entregas, não a calendário. Vocês não pagam o próximo bloco até receber o anterior em mãos.

"E se a equipe sumir?"

Código entregue a cada sprint no repositório de vocês. Documentação completa + treinamento da equipe inclusos.

"E se a tech ficar obsoleta?"

Stack mainstream (.NET 8, Vue 3, SQL Server) — qualquer fornecedor pega a continuidade. Sem lock-in técnico.

"E se um sócio sair?"

Banco de dados dedicado: exporta o banco inteiro dele e entrega. Sem trauma técnico, sem trauma jurídico.

Cada medo tem uma cláusula contratual ou técnica. Os riscos foram pensados antes — não depois do problema.
