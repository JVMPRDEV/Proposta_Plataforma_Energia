03 Arquitetura Técnica

Reverse Proxy Nginx
Real-time SignalR
Infraestrutura Docker + AWS (EC2/ECS + RDS)
Armazenamento de Imagens AWS S3 (templates, logos, QR Code, código de barras, prints da conta de luz — versionados por tenant/fatura)
Mensageria / Eventos AWS SNS + SQS FIFO + Lambda (eventos de domínio com ordem garantida, deduplicação por tenant+fatura e DLQ)

Padrões Arquiteturais

Monolito Modular

Aplicação única organizada em módulos independentes. Simplicidade de deploy com boa separação.

DDD (Domain-Driven Design)

Código organizado por domínio de negócio, com contextos delimitados bem definidos.

CQRS

NHibernate para escrita (commands) e Dapper para leitura (queries) - alta performance.

Clean Architecture

Camadas bem definidas com inversão de dependência.

Multitenancy Multibanco

Cada sócio com seu próprio banco de dados dedicado. Segurança, backup e restore totalmente independentes.

Event-Driven Architecture

Arquitetura orientada a eventos: módulos comunicam-se de forma desacoplada via Domain Events e Integration Events, permitindo extensibilidade sem reescrita.

Máquinas de Estado

Ciclos de vida críticos (Contrato, Fatura, Crédito, Cobrança) modelados como state machines explícitas — transições válidas, auditáveis e testáveis.

API REST + Swagger

Endpoints documentados automaticamente via OpenAPI/Swagger para fácil integração.

🧭 Decisão Arquitetural: Segregação Completa da Infraestrutura?

Uma pergunta legítima a ser feita pelos sócios: cada operação deve ter sua própria infraestrutura (banco, app, ambiente) totalmente isolada, ou todos compartilham a mesma plataforma com isolamento físico via multitenancy multibanco (1 database dedicado por sócio)?

✅ Pontos Positivos da Segregação Total

  • Isolamento absoluto — falha, deploy ou bug em uma operação não afeta as demais
  • Soberania total — cada sócio decide janela de manutenção, versão, backup, retenção
  • Compliance simplificado — dados nunca compartilham hardware com terceiros
  • Portabilidade — saída de um sócio não impacta os outros (basta entregar a stack dele)

⚠️ Pontos Negativos da Segregação Total

  • Custo de infra multiplicado — N ambientes em vez de 1 (banco, app, observabilidade, backups)
  • Manutenção N vezes maior — cada deploy, patch e upgrade replicado por sócio
  • Sem economia de escala — cada sócio paga o "piso fixo" da infraestrutura
  • Consolidação de indicadores torna-se um projeto de integração à parte
  • Suporte mais caro — horas de sustentação proporcionais ao número de ambientes

Recomendação técnica: a arquitetura proposta (Multitenancy Multibanco) já entrega isolamento físico total de dados — cada sócio tem seu próprio banco de dados dedicado, com backup e segurança totalmente independentes — preservando a economia de escala compartilhada da camada de aplicação. A segregação completa de infraestrutura permanece como opção, mediante orçamento adicional, caso algum sócio tenha exigência regulatória específica.

05 Investimento

ⓘ Modelo de Precificação Modular

Para reduzir a barreira de entrada e permitir flexibilidade, o investimento foi estruturado em 3 blocos independentes. Isso permite que um único sócio inicie o projeto com um MVP enxuto e adicione módulos conforme novos parceiros entrem ou a operação cresça.

BLOCO 1 Assessment Individual

⚠ Assessment cobrado à parte para TODOS os sócios

O Assessment é uma fase obrigatória e cobrada à parte para todos os sócios — inclusive a ESQ Energia (primeiro sócio). Não está incluso no valor do Core. O investimento varia entre R$ 2.500 e R$ 6.000 por sócio, conforme a complexidade da operação mapeada.

Escopo do Assessment: fase obrigatória cobrada por sócio. Consiste no mapeamento da operação, migração de dados e adaptação às regras específicas de cada parceiro — do primeiro ao último.

Assessment Individual

Mapeamento e migração por sócio

R$ 2.500 — 6.000

Valor variável conforme a complexidade da operação

  • Mapeamento de processos e rotinas atuais
  • Análise de planilhas e sistemas legados
  • Migração de dados (clientes, UCs, contratos)
  • Workshop de requisitos com stakeholders
  • Configuração de regras de rateio
  • Adaptação às particularidades do sócio

⚠ Importante: Operações de média/grande complexidade (múltiplas usinas, integrações com ERP, regras de negócio não-padrão, fluxos customizados) podem requerer orçamento de desenvolvimento individual, separado e adicional ao valor do Assessment, para implementar funcionalidades específicas que fogem do escopo do Core.

BLOCO 2 Desenvolvimento do Core da Plataforma

Investimento fixo e único para construir o motor principal da plataforma. Este é o "coração" do sistema que será compartilhado por todos os sócios, com isolamento físico de dados via Multitenancy Multibanco (1 banco de dados dedicado por sócio).

Intervenções técnicas — correções, manutenção evolutiva, novas integrações (Asaas, COELBA, WhatsApp) e novas features — são orçadas de forma avulsa e pontual, sob demanda, mediante escopo específico aprovado a cada chamado.

✓ Vantagem: você paga apenas pelo que efetivamente consome. Sem mensalidade travando caixa, sem horas não utilizadas evaporando no fim do mês. Cada demanda vira um orçamento transparente, com escopo e prazo fechados antes da execução.

Infraestrutura Elástica (Estimativa Mensal)

A infraestrutura na nuvem (AWS) é elástica e escala proporcionalmente ao tamanho da operação. Vocês começam enxutos no MVP e o custo cresce junto com o volume — sem licenças fixas que viram desperdício.

Infra + APIs (AWS + WhatsApp + processamento) R$ 200 ~ 1.150/mês (escala com uso e nº de sócios)
Banco SQL Server Express (gratuito) R$ 0/mês (limite 10 GB/banco — atende o início; ao escalar, edição paga ~R$ 1.200–1.800/mês por sócio)
Gateway Asaas (se contratado) Taxa por transação (pay-per-use)
API WhatsApp (se contratado) R$ 100 ~ 200/mês (variável por mensagem)
Certificado SSL Incluído

*Modelo pay-as-you-grow: a infra acompanha o crescimento da operação sem custo inicial inflado.

Modelo de Intervenções Pontuais

Não há pacotes mensais, fee fixo ou horas contratadas antecipadamente. Cada intervenção técnica pós Go-Live — seja correção de bug, manutenção evolutiva, integração com Asaas/COELBA, ajustes regulatórios ou desenvolvimento de novas features — é analisada, escopada e orçada de forma avulsa antes da execução.

A infraestrutura AWS (hospedagem, banco, gateway, WhatsApp API) é contratada e paga diretamente pelo cliente, sem markup nem repasse. O cliente mantém total controle sobre os custos recorrentes e só aciona o time técnico quando há demanda real.
